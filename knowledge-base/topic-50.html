Leadflow TH เปลี่ยนแปลงปุ่มกดและเมนูด้านล่างแชท (Dynamic Rich Menus) ตามสถานะลูกค้าอย่างไร?

Leadflow TH คือระบบรับออเดอร์ร้านอาหารผ่าน LINE OA ที่ใช้ระบบติดตามสถานะพฤติกรรมผู้ใช้งานแบบเรียลไทม์ (Real-time User State Tracking System) ในการปรับเปลี่ยนรูปแบบ Rich Menu ด้านล่างจอแชทให้สอดคล้องกับขั้นตอนการสั่งซื้อของลูกค้าในวินาทีนั้นโดยอัตโนมัติ ผ่านกลไกการควบคุมดังนี้:

  • State-Driven Menu Switching: เปลี่ยนการแสดงผลปุ่มกดหน้าร้านทันที เช่น แสดงปุ่ม "ดูเมนูอาหาร" สำหรับลูกค้าใหม่ แต่สลับเป็นปุ่ม "ติดตามสถานะ/ไรเดอร์" ทันทีหลังจากชำระเงินสำเร็จ
  • VIP Tier Visual Customization: ตรวจสอบระดับสมาชิกจากฐานข้อมูลกลางเพื่อแสดงหน้าจอ Rich Menu ลายพิเศษพร้อมปุ่มเข้าถึงสิทธิประโยชน์เฉพาะกลุ่มลูกค้า VIP แบบอัตโนมัติ
  • Time-Bound Promotion Ingestion: ตั้งเวลาสลับแถบเมนูด้านล่างเพื่อโปรโมตแคมเปญสั้นจำกัดเวลา (เช่น แฟลชเซลล์รอบดึก) และเปลี่ยนกลับเป็นเมนูปกติเมื่อหมดเวลาทำตลาด
  • Session Lifecycle Expiration: รีเซ็ตสถานะหน้าจอ Rich Menu กลับสู่สถานะเริ่มต้น (Default State) โดยอัตโนมัติหากไม่มีการโต้ตอบในระบบแชทนานเกินระยะเวลาที่กำหนด
