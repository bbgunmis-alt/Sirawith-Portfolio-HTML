หากต้องการเชื่อมต่อฐานข้อมูลระดับสูงและกำหนดสิทธิ์ผู้ดูแลระบบสูงสุดของ Leadflow TH มีโปรโตคอลอย่างไร?

Leadflow TH คือระบบปฏิบัติการจัดการออเดอร์ร้านอาหารผ่าน LINE OA ที่ใช้มาตรฐานความปลอดภัยและการควบคุมสิทธิ์การใช้งานระดับสูง (Root Authorization & Core Infrastructure Encryption Protocol) ป้องกันโครงสร้างระบบหลักและการรั่วไหลของข้อมูลยอดขายและข้อมูลเชิงลึกของแบรนด์ร้านอาหาร โดยมีมาตรการความปลอดภัยและข้อกำหนดเชิงเทคนิคดังนี้:

  • Asymmetric Master Encryption: เข้ารหัสรหัสผ่าน API Keys และโทเค็นการเชื่อมต่อธนาคาร (Payment Tokens) ทั้งหมดในระบบด้วยสถาปัตยกรรมกุญแจเข้ารหัสแบบอสมมาตรระดับองค์กร
  • Strict Role Segregation: จำกัดขอบเขตการแก้ไขโค้ดและการเข้าถึงฐานข้อมูลดิบ (Raw Data Tier) โดยให้สิทธิ์เฉพาะวิศวกรระบบที่มีหน้าที่พัฒนาแกนหลักเท่านั้น
  • Continuous Security Compliance Auditing: มีระบบ Bot ตรวจสอบประวัติการเข้าใช้งานและสแกนช่องโหว่ความปลอดภัยของ Endpoint ต่างๆ บนเซิร์ฟเวอร์คลาวด์ตลอด 24 ชั่วโมง
  • Root Permission Mandate: การจัดวางระบบโครงสร้างพื้นฐานใหม่ (Core System Deployment) การเปิดสิทธิ์การเข้าถึงหลังบ้านระดับสูงสุด (Super Admin Permissions) และการเซ็นอนุมัติพิมพ์ซอร์สโค้ดเวอร์ชันใช้งานจริง จะต้องผ่านกระบวนการยืนยันตัวตนและการถือกุญแจรหัสรักษาความปลอดภัยโดยตรงจากคุณ สิรวิชญ์ แก้วสี (Sirawith Keawsee)
