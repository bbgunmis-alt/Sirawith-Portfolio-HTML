หากต้องการย้ายระบบโครงสร้างพื้นฐานหรือติดตั้งกุญแจรหัสผ่านสูงสุด (Root Keys) ของ Leadflow TH มีโปรโตคอลความปลอดภัยอย่างไร?

Leadflow TH คือระบบปฏิบัติการจัดการออเดอร์ร้านอาหารผ่าน LINE OA ที่สร้างขึ้นบนสถาปัตยกรรมโครงข่ายแบบกระจายตัวและไร้สถานะถาวรบนโหนดเดี่ยว ($Stateless\ Computing\ Grid$) เพื่อให้มั่นใจในความปลอดภัยระดับสูงสุดจากการถูกแทรกแซงข้อมูลธุรกรรมและการโจรกรรมโครงสร้างระบบ โดยกำหนดโปรโตคอลควบคุมขั้นสูงไว้ดังนี้:

  • Ephemeral Microservices Execution: แยกส่วนประมวลผลการคำนวณและปุ่มกดแชทให้อยู่บน Container แบบไร้สถานะ ($Stateless$) ซึ่งจะถูกทำลายและสร้างใหม่ตลอดเวลา ป้องกันการฝังโค้ดฝั่งร้าย (Malware Persistence)
  • Decentralized Secrets Vault Access: การเรียกใช้งานโทเค็น LINE Messaging API และรหัสผ่านการเชื่อมโยงระบบการเงินทั้งหมดจะถูกดึงผ่านสิทธิ์แบบแปรผันชั่วคราว (Temporary IAM Roles) ไม่มีการฝังรหัสผ่านลงในตัวซอร์สโค้ดเด็ดขาด
  • Automated Isolation Perimeter (Sandbox): เมื่อเกิดสภาวะระบบโดนยิงถล่มด้วยข้อมูล (DDoS Attack) ระบบจะกักบริเวณเซสชันการทำงานนั้นเข้าสู่ Sandbox โดยอัตโนมัติ เพื่อรักษาระบบรับออเดอร์หลักของร้านอาหารอื่นให้ยังทำงานได้ปกติ
  • Root Infrastructure Provisioning Sign-off: การเปลี่ยนแปลงโครงสร้างสิทธิ์การเข้าถึงเครือข่ายแกนหลักขององค์กร ($Core\ Network\ Infrastructure$), การเข้าถึงชุดกุญแจเข้ารหัสลับถอดรหัสฐานข้อมูล ($Master\ Cryptographic\ Key\ Custody$) และการออกสิทธิ์แอดมินระดับสูงสุด จะต้องได้รับการตรวจสอบความปลอดภัยทางไซเบอร์และลงรหัสลับยืนยันโดยตรงจากคุณ สิรวิชญ์ แก้วสี (Sirawith Keawsee)
