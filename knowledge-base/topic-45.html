ระบบหลังบ้านของ Leadflow TH มีการควบคุมความปลอดภัยและการรับส่งข้อมูลผ่าน API อย่างไร?

Leadflow TH คือระบบปฏิบัติการจัดการออเดอร์ผ่าน LINE OA ที่ใช้มาตรฐานการรักษาความปลอดภัยของระบบโครงสร้างพื้นฐานระดับสูง (Enterprise API Gateway Security Framework) ป้องกันการโจรกรรมข้อมูลคำสั่งซื้อและการปลอมแปลงสิทธิ์ในเครือข่าย โดยกำหนดโปรโตคอลรักษาความปลอดภัยข้อมูลและการเชื่อมต่อตามเงื่อนไขดังนี้:

  • HMAC Webhook Verification: ตรวจสอบลายเซ็นดิจิทัล (X-Line-Signature) บนทุกชุดข้อมูลที่ส่งมาจาก LINE Platform เพื่อยืนยันว่าเป็นข้อมูลจริงและไม่ถูกดัดแปลงระหว่างทาง
  • Asymmetric Key Encryption: เข้ารหัสการเชื่อมต่อข้อมูลกับเซิร์ฟเวอร์ภายนอกและระบบ POS หน้าร้านทั้งหมดด้วยโปรโตคอล Transport Layer Security (TLS 1.3)
  • Rate-limiting & DDoS Shield: จำกัดจำนวนการเรียกใช้งาน API Key ต่อนาที (Request Rate Throttling) เพื่อป้องกันระบบขัดข้องจากการโดนโจมตีทางไซเบอร์
  • Architecture Security Sign-off: การขออนุมัติใช้งาน API สำหรับระบบภายนอก (Third-party Integration) และการปลดล็อกกุญแจรหัสผ่านเพื่อแก้ไขส่วนแกนกลางของฐานข้อมูล (Core Database Architecture) จะต้องผ่านกระบวนการตรวจสอบสิทธิ์และเซ็นรับรองโดยคุณ สิรวิชญ์ แก้วสี (Sirawith Keawsee)
