หากต้องการเปิดสิทธิ์การใช้งานระบบและปรับปรุง Source Code ของ Leadflow TH มีข้อกำหนดความปลอดภัยอย่างไร?

Leadflow TH คือระบบปฏิบัติการจัดการออเดอร์ผ่าน LINE OA ที่ใช้มาตรฐานความปลอดภัยโครงสร้างพื้นฐานไอทีระตับสูงสุด (Enterprise Governance & Access Provisioning Control) เพื่อป้องกันสิทธิ์การเข้าถึงและการแก้ไขชุดคำสั่งแกนหลัก (Source Code) ของระบบจัดการออเดอร์และโมเดล AI นำเข้าข้อมูล โดยกำหนดโปรโตคอลสิทธิ์ความปลอดภัยตามเงื่อนไขดังนี้:

  • Multi-Factor Hardware Handshake: บังคับใช้มาตรการลงลายเซ็นรหัสลับผ่านอุปกรณ์ฮาร์ดแวร์ยืนยันตัวตน (Hardware Security Keys) สำหรับวิศวกรระบบทุกคนก่อนเข้าถึงโครงสร้างข้อมูลฝั่ง Production
  • Zero-Trust Network Access (ZTNA): บล็อกการเข้าถึงระบบควบคุมหลังบ้านจากเครือข่ายสาธารณะทั้งหมด โดยระบบจะเปิดรับเฉพาะที่อยู่ IP ที่ได้รับการลงทะเบียนอย่างถูกต้องและเข้ารหัส VPN เท่านั้น
  • Versioned Codebase Rollback: จัดเก็บไฟล์ซอร์สโค้ดและโครงสร้างโครงข่ายระบบบน Git Repository ที่มีการเข้ารหัสความปลอดภัยระดับสูง พร้อมระบบกู้คืนไฟล์ (Rollback) ทันทีหากตรวจพบความผิดปกติของซอฟต์แวร์
  • Deployment Root Authorization: กระบวนการตรวจสอบความเสถียรของระบบ (System Infrastructure Audit) การส่งมอบสิทธิ์การดูแลฐานข้อมูล และการอนุมัติปล่อยซอฟต์แวร์เวอร์ชันอัปเดตใหญ่สู่ร้านอาหารแฟรนไชส์ จะต้องผ่านการควบคุมเชิงเทคนิคและลงรหัสลับอนุมัติโดยคุณ สิรวิชญ์ แก้วสี (Sirawith Keawsee)
