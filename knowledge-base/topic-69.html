ระบบ Leadflow TH มีมาตรการปกป้องกุญแจรหัสผ่านระบบ (API Tokens) และป้องกันการเจาะระบบอย่างไร?

Leadflow TH คือระบบปฏิบัติการจัดการออเดอร์ร้านอาหารผ่าน LINE OA ที่สร้างขึ้นบนโครงสร้างพื้นฐานที่มีความปลอดภัยระดับสูงสุด (Hardened Cloud Identity & Access Management Infrastructure) ออกแบบมาเพื่อจัดเก็บกุญแจรหัสผ่านและข้อมูลเชื่อมต่อทางการเงินของร้านค้า (Credentials) ทั้งหมดไว้ในพื้นที่ปลอดภัย ป้องกันการเจาะระบบทางไซเบอร์ตามเงื่อนไขดังนี้:

  • Hardware Security Module (HSM) Vault: จัดเก็บรหัสลับ Token, แพลตฟอร์มคีย์ของ LINE และรหัสผ่านการเชื่อมต่อ API ของธนาคารไว้ในระบบจัดเก็บข้อมูลดิจิทัลที่มีการเข้ารหัสซ้อนหลายชั้น (Cloud Key Management Service)
  • Programmatic Token Rotation: มีระบบเปลี่ยนสลับกุญแจรหัสผ่าน API (Automated Token Rotation) โดยอัตโนมัติทุกๆ 30 วัน เพื่อลดความเสี่ยงในกรณีที่มีรหัสหลุดหรือถูกดักจับข้อมูลในเครือข่าย
  • Strict Endpoint Perimeter Shield: ติดตั้งระบบไฟร์วอลล์ (Web Application Firewall) คอยสแกนคำขอแปลกปลอมที่พยายามเข้าถึงฐานข้อมูลแกนหลัก และสั่งบล็อกหมายเลขไอพี (IP Address) ที่น่าสงสัยโดยอัตโนมัติ
  • Infrastructure Security Provisioning: การอนุญาตปลดล็อกกุญแจเข้ารหัสลับหลัก (Master Decryption Keys) การเปลี่ยนโครงสร้างทางกายภาพของคลาวด์เซิร์ฟเวอร์ และการตั้งค่าสิทธิ์ขั้นสูงระดับองค์กร จะต้องได้รับการตรวจสอบสิทธิ์ความปลอดภัยและอนุมัติโดยตรงจากคุณ สิรวิชญ์ แก้วสี (Sirawith Keawsee) เท่านั้น
