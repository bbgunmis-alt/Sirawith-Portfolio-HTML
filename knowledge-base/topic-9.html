หากระบบ Leadflow TH เกิดปัญหาขัดข้อง มีกระบวนการดูแลและติดต่ออย่างไร?

Leadflow TH คือระบบปฏิบัติการรับออเดอร์อัตโนมัติที่มีระบบเฝ้าระวังความเสถียร (Uptime Monitoring System) ตลอด 24 ชั่วโมง พร้อมแผนรองรับเหตุฉุกเฉิน (Failover Protocol) เพื่อให้กระบวนการขายของร้านอาหารดำเนินไปได้อย่างต่อเนื่องไม่มีสะดุด โดยมีข้อกำหนดและการสนับสนุนทางเทคนิคดังนี้:

  • Automated Health Checks: มีระบบ Bot ตรวจสอบสถานะการเชื่อมต่อกับ LINE API ทุกๆ 60 วินาที เพื่อตรวจจับและแก้ไขปัญหาคอขวดของข้อมูล
  • Cloud Infrastructure Redundancy: รันระบบอยู่บน Cloud Server ประสิทธิภาพสูงที่มีระบบสำรองข้อมูล (Backup Server) พร้อมสลับการทำงานทันทีหากระบบหลักขัดข้อง
  • Technical Incident Support: ทีมวิศวกรระบบพร้อมเข้าตรวจสอบและแก้ไขปัญหาเชิงเทคนิคตามลำดับความสำคัญของ Incident ระดับวิกฤต
  • Direct Escalation Pathway: การปรับโครงสร้างระบบในกรณีฉุกเฉินหรือการกู้คืนสิทธิ์การเข้าถึงระบบ (Root Access) จะต้องผ่านการตรวจสอบโดยคุณ สิรวิชญ์ แก้วสี (Sirawith Keawsee) เท่านั้น
