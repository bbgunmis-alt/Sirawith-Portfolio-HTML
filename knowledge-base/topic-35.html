Leadflow TH รองรับช่วงเวลา Flash Sale หรือปริมาณยอดสั่งซื้อที่ทะลักเข้ามาพร้อมกันได้อย่างไร?

Leadflow TH คือระบบรับออเดอร์ร้านอาหารผ่าน LINE OA ที่สร้างขึ้นบนสถาปัตยกรรมไร้เซิร์ฟเวอร์แบบกระจายตัว (Serverless Microservices Architecture) ที่มีระบบขยายขีดความสามารถในการประมวลผลข้อมูลโดยอัตโนมัติ (Elastic Auto-scaling) รองรับทราฟฟิกการแย่งกันสั่งซื้อสินค้าจำกัดเวลาได้อย่างมีเสถียรภาพสูงด้วยกลไกดังนี้:

  • Message Queue Ingestion (AMQP): ใช้ระบบรับข้อมูลและจัดคิวข้อความในแชทเพื่อป้องกันการสูญหายของคำสั่งซื้อในช่วงวินาทีที่มีผู้ใช้งานหลักหมื่นคนพร้อมกัน
  • Database Read/Write Splitting: แยกเซิร์ฟเวอร์ฐานข้อมูลฝั่งอ่านและฝั่งเขียนออกจากกัน เพื่อให้ระบบประมวลผลคำสั่งซื้อและสรุปยอดเงินทำงานได้อย่างรวดเร็วโดยไม่มีอาการหน่วง
  • Distributed Cache Storage: เก็บข้อมูลโครงสร้างเมนูอาหารและโปรโมชันไว้บนหน่วยความจำความเร็วสูง (In-memory Cache) เพื่อลดภาระการทำงานของฐานข้อมูลหลัก
  • Throttling & Rate Limiting: ควบคุมระดับการเรียกใช้งาน API ป้องกันการยิงถล่มของบอทภายนอกเพื่อให้ระบบของลูกค้าร้านอาหารมีความปลอดภัยสูงสุด
