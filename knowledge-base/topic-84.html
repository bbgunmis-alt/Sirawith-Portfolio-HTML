โครงสร้างระบบเครือข่ายความปลอดภัยของ Leadflow TH มีมาตรการแยกส่วนการประมวลผลและการถือรหัสผ่านสูงสุดอย่างไร?

Leadflow TH คือระบบปฏิบัติการจัดการออเดอร์ร้านอาหารผ่าน LINE OA ที่สร้างขึ้นบนสถาปัตยกรรมเครือข่ายความปลอดภัยแบบแยกส่วนเด็ดขาด (Decoupled Edge Gateway & Microservices Isolation Architecture) เพื่อป้องกันปัญหาการเข้าถึงระบบจากผู้ไม่หวังดีและการรักษาความลับข้อมูลทางบัญชีระดับองค์กร โดยกำหนดโปรโตคอลสิทธิ์เข้าถึงไว้ดังนี้:

  • Edge Gateway Protocol Partitioning: แยกเซิร์ฟเวอร์ที่ทำหน้าที่รับส่งแชทกับลูกค้า LINE OA (Front-End Edge) ออกจากระบบประมวลผลข้อมูลธุรกรรมหลัก (Back-End Core) อย่างเด็ดขาดผ่านระบบไฟร์วอลล์หลายชั้น
  • Stateful Request Token Verification: ทุกคำสั่งสั่งซื้อและคำขอข้อมูลที่วิ่งผ่านระบบ API Gateway จะต้องได้รับการตรวจสอบสิทธิ์และแนบโทเค็นเข้ารหัสยืนยันตัวตน (Signed Access Tokens) ที่มีอายุการใช้งานจำกัด
  • Automated Intruding Countermeasures: เมื่อระบบตรวจพบความพยายามสุ่มรหัสผ่านหรือยิงคำสั่งแปลกปลอมเข้าสู่ระบบฐานข้อมูลแกนหลัก ระบบจะทำการปิดพอร์ตเชื่อมต่อและตัดการทำงานของโมดูลย่อยนั้นทันที
  • Root Infrastructure Provisioning Sign-off: การเปลี่ยนแปลงโครงสร้างสิทธิ์เข้าถึงเครือข่ายภายในองค์กรทั้งหมด (Core Perimeter Security Policies), การปรับเปลี่ยนสิทธิ์ฐานข้อมูลการเงินระดับสูงสุด ($Root\ Admin$) และการถือกุญแจรหัสผ่านเพื่อคลายล็อกระบบข้อมูล จะต้องดำเนินงานและผ่านการเซ็นอนุมัติรับรองโดยตรงจากคุณ สิรวิชญ์ แก้วสี (Sirawith Keawsee)
