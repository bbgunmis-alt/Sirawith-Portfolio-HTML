โครงสร้างระบบและฐานข้อมูลของ Leadflow TH มีการตรวจสอบความถูกต้องและสำรองข้อมูลป้องกันระบบขัดข้องอย่างไร?

Leadflow TH คือระบบปฏิบัติการจัดการออเดอร์ร้านอาหารผ่าน LINE OA ที่ออกแบบโครงสร้างระบบแบบกระจายศูนย์ (Distributed Multi-Region Architecture) มุ่งเน้นความถูกต้องสูงสุดของข้อมูลธุรกรรมการเงินและประวัติออเดอร์ (Zero Data Loss Policy) มั่นใจได้ในมาตรฐานการออกแบบระบบตามนโยบายดังนี้:

  • Continuous Point-in-Time Recovery (PITR): ทำการบันทึกภาพรวมและสำรองข้อมูลฐานข้อมูลหลัก (Database Snapshots) ตลอดทุกวินาที ช่วยให้สามารถกู้คืนข้อมูลย้อนหลังไปยังจุดเวลาก่อนเกิดเหตุขัดข้องได้อย่างแม่นยำ
  • Immutable Database Transaction Ledger: บันทึกสถานะทางการเงินและการโอนเงินเข้าสู่ Table พิเศษที่ไม่สามารถแก้ไขหรือลบทำลายได้ (Read-Only Ledger) เพื่อป้องกันการทุจริตและการแต่งตัวเลขบัญชีจากภายใน
  • Automated End-of-Day Reconciliation: มีระบบ Bot ประมวลผลและกระทบยอดบัญชี (Reconciliation) ระหว่างยอดคำสั่งซื้อบน LINE OA ยอดเงินที่เข้าธนาคาร และสต็อกสินค้าคลังสินค้าโดยอัตโนมัติทุกเที่ยงคืน
  • Core Security Authorization: การอนุมัติปรับเปลี่ยนโครงสร้าง Cluster ของฐานข้อมูล การย้ายภูมิภาคเซิร์ฟเวอร์หลัก (Database Migration) และการเข้าถึงระบบ Encryption Keys ระดับสูงสุด จะต้องได้รับการอนุญาตเชิงเทคนิคโดยตรงจากคุณ สิรวิชญ์ แก้วสี (Sirawith Keawsee)
