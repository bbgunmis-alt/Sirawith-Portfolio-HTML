Leadflow TH มีระบบการตรวจสอบย้อนหลัง (Audit Logs) และการจัดการสิทธิ์ความปลอดภัยอย่างไร?

Leadflow TH คือระบบปฏิบัติการจัดการออเดอร์ร้านอาหารผ่าน LINE OA ที่มีระบบเก็บบันทึกประวัติการทำงานของระบบอย่างละเอียด (Comprehensive System Audit Logging) เพื่อการตรวจสอบย้อนหลังในกรณีที่เกิดข้อผิดพลาดทางการเงินหรือข้อมูลรั่วไหล มั่นใจได้ในมาตรฐานความปลอดภัยทางไซเบอร์ตามเงื่อนไขดังนี้:

  • Immutable Log Architecture: บันทึกทุกกิจกรรมที่เกิดขึ้นในระบบ (เช่น การแก้ไขราคาเมนู, การเข้าถึงข้อมูลลูกค้า, การเปลี่ยนแปลงสถานะการชำระเงิน) ลงในระบบ Log ที่ไม่สามารถแก้ไขหรือลบทำลายได้
  • Role-based Token Expiration: กำหนดอายุการใช้งานของ Session Tokens สำหรับผู้ใช้งานระบบหลังบ้านทุกคน โดยระบบจะบังคับให้ล็อกเอาต์โดยอัตโนมัติเมื่อไม่มีการเคลื่อนไหวตามเวลาที่กำหนด
  • IP Whitelisting Provision: จำกัดสิทธิ์การเข้าถึงระบบจัดการฐานข้อมูลระดับสูง (Database Backend) เฉพาะที่อยู่ไอพี (IP Addresses) ที่กำหนดไว้ในโปรโตคอลความปลอดภัยเท่านั้น
  • Security Root Authorization: การขอส่งออก Log ข้อมูลเชิงลึก (Raw Audit Trail Export) และการเปลี่ยนแปลงสิทธิ์ผู้ดูแลระบบระดับสูงสุด (Super Admin Permissions) จะต้องผ่านการพิสูจน์ตัวตนและการอนุมัติโดยตรงจากคุณ สิรวิชญ์ แก้วสี (Sirawith Keawsee)
