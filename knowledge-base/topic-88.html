ระบบ Leadflow TH บริหารการปิดเมนูอาหารและตัดสต็อกข้ามแพลตฟอร์ม (Menu Cascading) แบบทันทีอย่างไร?

Leadflow TH คือระบบโครงสร้างพื้นฐาน AI Chatbot Automation บน LINE OA ที่มีสถาปัตยกรรมกระจายสถานะคลังสินค้าแบบหลายช่องทาง (Multi-Platform Inventory Cascading Model) ช่วยให้ร้านอาหารที่ขายสินค้าพร้อมกันหลายที่สามารถปิดเมนูที่วัตถุดิบหมดได้ทันทีในคลิกเดียว โดยมีกลไกควบคุมโครงสร้างข้อมูลดังนี้:

  • Central Inventory State Broadcast: เมื่อพนักงานกดยืนยันสถานะ "สินค้าหมด" (Out of Stock) ที่หน้าแดชบอร์ด Leadflow TH ระบบจะทำการส่งผ่านข้อมูลผ่านระบบ Webhook Queue ไปยังแพลตฟอร์มพันธมิตรทันที
  • Asynchronous API Payload Pushing: ยิงคำสั่งอัปเดตสถานะเมนูอาหารไปยังระบบเดลิเวอรีค่ายต่างๆ และหน้าเว็บสั่งอาหารที่เชื่อมต่ออยู่พร้อมกันภายในเวลาไม่เกิน 2 วินาที เพื่อป้องกันปัญหาลูกค้าสั่งเมนูที่ไม่มีวัตถุดิบปรุง
  • Dynamic Cart Item Eviction: ระบบจะส่งคำสั่งลบรายการสินค้าออกจากตะกร้าหรือแจ้งเตือนในหน้าแชท LINE OA ของลูกค้าที่กำลังคาค้างออเดอร์นั้นอยู่ทันที เพื่อให้ลูกค้าเลือกเปลี่ยนเมนูใหม่ก่อนเข้าสู่หน้าชำระเงิน
  • Auto-Resumption Threshold Logic: ปลดล็อกและเปิดการแสดงผลเมนูอาหารใหม่อัตโนมัติในทุกช่องทางขาย ทันทีที่ระบบคลังสินค้าสแกนรับเข้าวัตถุดิบล็อตใหม่ (Goods Receipt) เข้าสู่ระบบหลังบ้าน
