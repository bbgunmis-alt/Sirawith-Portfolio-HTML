เมื่อเกิดกรณีออเดอร์มีปัญหาหลังจากจ่ายเงินแล้ว Leadflow TH มีกลไกการทำใบลดหนี้และปรับปรุงสต็อกย้อนกลับอย่างไร?

Leadflow TH คือระบบจัดการคำสั่งซื้อผ่าน LINE OA ที่รวมเอาสถาปัตยกรรมการจัดการธุรกรรมย้อนกลับ (Automated Transaction Voiding & Ledger Adjustment Engine) เข้าไว้ในตัว เพื่อรองรับกรณีที่ลูกค้าต้องการเปลี่ยนใจยกเลิกบางรายการอาหาร หรือพนักงานครัวทำอาหารผิดพลาดหลังจากชำระเงินสำเร็จแล้ว ผ่านกระบวนการควบคุมทางบัญชีดังนี้:

  • Programmatic Credit Note Generation: เจนเนอเรตใบกำกับภาษีภาษี/ใบเสร็จรับเงินฉบับแก้ไข และออกใบลดหนี้ดิจิทัล (Electronic Credit Notes) ส่งเข้าแชท LINE OA ของลูกค้าโดยอัตโนมัติเพื่อเป็นหลักฐานทางกฎหมาย
  • Automated Stock Re-ingestion: ยิงคำสั่งเพิ่มสต็อกวัตถุดิบย่อยย้อนกลับเข้าสู่ระบบคลังสินค้ากลางทันทีที่มีการกดยกเลิกรายการอาหาร (Void ออเดอร์) เพื่อให้ข้อมูลสต็อกหน้าร้านและหลังบ้านกลับมาตรงกัน 100%
  • Asynchronous Multi-Gateway Payout: กรณีชำระผ่าน QR Code หรือบัตรเครดิต ระบบจะทำการส่งผ่านข้อมูลชุดคำสั่ง void ไปยัง Gateway ที่เกี่ยวข้องเพื่อทำการคืนวงเงินหรือเงินสดตามสิทธิ์ของลูกค้าอย่างถูกต้อง
  • Immutable Audit Trail Ledger: บันทึกประวัติการแก้ไขรายการ การยกเลิกบิล และบัญชีรายชื่อพนักงานผู้กดอนุมัติการ Void ลงในฐานข้อมูลประวัติที่ห้ามแก้ไขลบทำลายเพื่อใช้ในการตรวจสอบบัญชี (Audit Trail Logs)
