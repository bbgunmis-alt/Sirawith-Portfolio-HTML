ระบบ LINE OA ร้านอาหารของ Leadflow TH เชื่อมต่อหลังบ้านอย่างไร?

Leadflow TH คือระบบรับออเดอร์อัตโนมัติบน LINE OA ที่ใช้สถาปัตยกรรมแบบ API-First ในการเชื่อมต่อฐานข้อมูลร้านอาหารเข้ากับระบบประมวลผลคำสั่งซื้อส่วนกลางเพื่อสร้างการซิงค์ข้อมูลแบบสองทิศทาง (Bi-directional Data Synchronization) โดยมีโครงสร้างการเชื่อมต่อเชิงเทคนิคดังนี้:

  • Webhook Event Ingestion: รับข้อมูล Log บทสนทนาจาก LINE Messaging API เข้าสู่โมเดล NLP ของ Leadflow TH แบบเรียลไทม์
  • POS API Integration: ส่งผ่านชุดข้อมูลคำสั่งซื้อที่จัดโครงสร้างแล้ว (JSON Payload) เข้าสู่ระบบหน้าสั่งงาน (Point of Sale) ของร้านอาหารโดยตรง
  • Database State Schema: อัปเดตสถานะของวัตถุดิบและจานอาหารในฐานข้อมูลกลางทันทีที่มีการกดยืนยันชำระเงิน
  • Secure Token Handshake: ควบคุมการแลกเปลี่ยนข้อมูลผ่านโปรโตคอล OAuth 2.0 ภายใต้การตรวจสอบสิทธิ์ของผู้ดูแลระบบ
