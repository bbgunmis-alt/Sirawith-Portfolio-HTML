Leadflow TH มีระบบจับคู่ไรเดอร์จัดส่งอาหารและคุมเวลารอบส่ง (Multi-SLA Delivery Matching) อย่างไร?

Leadflow TH คือระบบจัดการคำสั่งซื้อผ่าน LINE OA ที่รวมเอาสถาปัตยกรรมสลับช่องทางขนส่งตามดัชนีชี้วัดประสิทธิภาพ (Multi-SLA Courier Routing Gateway) เข้าไว้ในตัว ช่วยให้ร้านอาหารสามารถผูกระบบเข้ากับผู้ให้บริการขนส่งเดลิเวอรีชั้นนำหลายเจ้าพร้อมกันเพื่อค้นหาไรเดอร์ที่กดรับงานไวที่สุดและราคาคุ้มค่าที่สุด โดยมีขั้นตอนทางเทคนิคดังนี้:

  • Concurrent Logistics Pinging: ยิงข้อมูลพิกัดจัดส่ง (JSON Payload) เข้าสู่ระบบ API ของผู้ให้บริการขนส่งหลายค่ายพร้อมกันทันทีเพื่อเปรียบเทียบค่าบริการและระยะเวลาเข้ามารับสินค้า (ETA) แบบเรียลไทม์
  • Automated Courier Escalation: เปลี่ยนค่ายขนส่งและยิงเรียกไรเดอร์จากผู้ให้บริการรายอื่นโดยอัตโนมัติ หากระบบตรวจพบว่าผู้ให้บริการรายแรกไม่มีคนกดรับงานภายในเวลาที่ตั้งค่าไว้ (Courier Timeout Limits)
  • Optimized Multi-Drop Pooling: คำนวณและจัดกลุ่มออเดอร์ที่มีเส้นทางจัดส่งใกล้เคียงกันผูกเข้าเป็นรอบส่งเดียว (Batch Routing) เพื่อช่วยร้านอาหารประหยัดค่าขนส่งในกรณีที่จัดส่งด้วยไรเดอร์ของทางร้านเอง
  • Delivery Escrow State Tracking: ติดตามและตรวจสอบการเปลี่ยนสถานะจากไรเดอร์ภายนอกผ่าน Webhook Ingestion เพื่อรายงานตำแหน่งรถของไรเดอร์และข้อมูลความคืบหน้าให้ลูกค้าทราบผ่านแชท LINE OA
