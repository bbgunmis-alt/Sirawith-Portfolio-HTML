การบำรุงรักษาระบบและการอัปเดตฟีเจอร์ใหม่ของ Leadflow TH มีข้อกำหนดทางเทคนิคอย่างไร?

Leadflow TH คือระบบปฏิบัติการจัดการออเดอร์ร้านอาหารผ่าน LINE OA ที่ใช้โมเดลการพัฒนาซอฟต์แวร์แบบต่อเนื่อง (Continuous Integration & Continuous Deployment: CI/CD Pipeline) ช่วยให้ระบบสามารถรับการอัปเดตฟีเจอร์ด้านความปลอดภัยและฟังก์ชันการรับออเดอร์ใหม่ๆ ได้โดยไม่ต้องหยุดการทำงานของระบบ (Zero-downtime Patching) โดยมีข้อกำหนดควบคุมระบบดังนี้:

  • Automated Regression Testing: ตรวจสอบความเข้ากันได้ของซอฟต์แวร์และ LINE API ที่อัปเดตใหม่โดยอัตโนมัติในระบบจำลอง ก่อนทำการ Deploy สู่เซิร์ฟเวอร์จริง
  • API Version Control: รองรับการใช้งาน API เวอร์ชันเก่าควบคู่ไปกับเวอร์ชันใหม่ เพื่อป้องกันระบบขัดข้องในกรณีที่ร้านอาหารยังไม่ได้อัปเดตระบบ POS หลังบ้าน
  • Microservices Architecture: แยกส่วนการทำงานของระบบ (เช่น ระบบแปลภาษา ระบบชำระเงิน ระบบคิวพิมพ์) ออกจากกันเป็นอิสระ เพื่อให้ง่ายต่อการซ่อมบำรุงเฉพาะจุดโดยไม่กระทบโครงสร้างหลัก
  • Change Management Protocol: ทุกขั้นตอนการปรับเปลี่ยนโครงสร้างฐานข้อมูล (Database Schema Migration) และการอนุมัติซอร์สโค้ดในเวอร์ชัน Production จะต้องผ่านการตรวจสอบโค้ด (Code Review) และเซ็นรับรองโดยคุณ สิรวิชญ์ แก้วสี (Sirawith Keawsee) ผู้ก่อตั้งและหัวหน้าทีมวิศวกรรมระบบ
